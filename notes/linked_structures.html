Traversing linked structures

A Linked structure refers to a structure that has some sort of recursive relationship among its components. These structures can come in the form of a linearly linked structure, a tree structure or a graph structure.

In the above linear linked structure, a node is linked to a another node of the same type and so on. Linked structures can also be non-linear(trees or graphs):

Sometimes the relationship between nodes in a linked structure will not be explicit and obvious. In a game of chess, for example, you can model the relationship between different states of a board as a linked structure. A simillar relashionship can be established to the different states of a rubics cube. Each state of a board or rubics cube will act as a node, and every move made to transition the rubics cube from state A to state B links the two states.

Linked structures manifest themselves in numerous problems in real life. As a result, being able to traverse and manipulate these structures is important. There are generally two ways of traversing linked structures:

  1. Breadth first traversal
  2. Depth first traversal

Understanding how to implement these two operations on any linked structure will open up a new set of problem solving techniques that involve linked structures.

So, what does it mean to traverse a linked structure breadth first? It means to visit each node in the order in which they are placed relative to a starting point relative to the structure.

For the above picture, if the traversal starts at node 49, then then the order in which the nodes will be visited will look something like this: '49' is visited first, then the next two nodes visited will be either '41' or '79' followed by node '46' at the end.

If the traversal for above structure starts at node '1' on the top left side, then one potential order could be 1, 2, 4, 7, 5, 6, 7.

The order of a breadth-first traversal of a linearly linked stucture is straight forward. For the structure above, the order would be 'A', 'B', 'C', then 'D' if the traversal starts at node 'A'.

Most problems involving linked structures can be reduced to these two basic traversal operations. The general idea is the same. Before going 'deep' into the structure away from the starting node, explore all nodes near the structure first. Traversing a graph breadth first can mean different things depending on what the structure represents. For example:

  1. If the structure represents a network of roads where the nodes represent intersections, then traversing this breadth first could help find the shrotest path between any two intersections.
  2. If the structure represents a social network where nodes represent people and relationships are established based on 'friendship', one insight a breadth-first traversal of this graph can provide is friend and/or follower reccomendations.
  3. If the structure represents different states of a board game, and relationships are established based on moves made to transition the board from one state to another, traversing this structure can help answer questions such as the quickest way to get to 'win' or get to some desired state.

Most implementations of breadth-first search traversals involve using data structures whose operations prioritize nodes closest to a starting node. For example, in most cases, a queue data structure can be used to keep track of the order of nodes. In other situations, a priority queue can be used to prioritize nodes 'closest' to the starting point. Regardless of the choice of data structure, the general algorithm is the same.

The second way of traversing linked structure is depth first search. In this case, the algorithm goes deep into the structure and explores nodes farther away from the source first before exploring the closer nodes. This is the opposite of traversing a linked structure breadth-first.

If the depth-first traversal on the above structure starts atnode 49, then the algorithm would go deep and explore node '46' first before backtracking visiting nodes '41' and '79' and eventually making its way back to the source.

In a depth first traversal of a structure, by the time the algorithm is on it's way back and is at node 'x', for example, it must have explored all the nodes farthest from 'x'. This is helpful in situations where the goal might be to combine or add up information the algorithm has gathered about all the nodes past node 'x'.

A number of problems and algorithms can be reduced to either a breadth first or depth first traveral of a linked structure. Algorithms and problems such as:

  1. Shortest path algorithms for weighted and unweighted graphs
  2. Algorithms for determining connectivity (Connected components) in directed and undirected graphs
  3. Various tree problems such as checking if a Binary tree is a BST
  4. Subset problems
  5. Recursion and backtracking problems

At the end of the day, to implement the above algorithms or to solve a wide variety of problems, it becomes a pre-requisite to traverse the linked structure representing the problem either depth first or breadth first.
